## 랜덤 식단 룰렛 MVP + Claude Code 확장 구축 완료

"오늘 뭐 먹지?" 랜덤 식단 룰렛을 학습용 단일 HTML 파일로 만들고, 이를 소재로 Claude Code 확장 3종(스킬·서브에이전트·훅)을 구축한 뒤 서브에이전트 리뷰로 검증·수정까지 한 묶음으로 진행했다.

| 항목 | 파일 |

### 핵심 구현 포인트

- **룰렛 좌표계 정합성**: `drawWheel()`(`-Math.PI/2`, 12시 시작)과 `spin()`의 `winnerCenter`가 모두 12시 기준 시계방향으로 일치. "화면에 멈춘 칸 = 표시 결과" 불변식 유지.
- **기획 재설계**: 1차로 localStorage·최근제외 등 풀기능 버전을 만들었으나, 학습 목적(개념 명확성 우선)에 맞춰 최소 범위로 재작성 후 카테고리 필터·히스토리만 점진 추가.
- **필터 UX 전환**: 라디오/단일선택 → `Set` 기반 체크박스 다중 선택(한식/일식/중식/양식 동시 선택, 최소 1개 유지).
- **Claude Code 확장 연동**: code-review 스킬 ← code-reviewer 서브에이전트가 호출하는 구조. 신규 스킬·에이전트는 세션 재시작 후 등록되는 특성 확인.
- **버그 수정 (서브에이전트 리뷰 발견, 교차검증 후 채택)**:
  - 🔴 스핀 도중 체크박스 변경 시 `transition`이 끊겨 `transitionend` 미발화 → `spinning=true` 영구 고착. `onchange` 첫머리에 스핀 중 변경 차단 가드 추가.
  - `transitionend` 리스너를 `{ once: true }`로 전환(누수·중복 방지).
  - 필터 변경 시 이전 당첨 결과 텍스트 초기화(룰렛/결과 불일치 방지).

## git commit 시 작업일지 자동화 훅 구축 + 첫 커밋/푸시

커밋 시 작업 내역을 함께 남기도록 강제하는 유저 레벨 PreToolUse 훅을 만들고, 그 훅을 통과시키며 변경분을 원격에 처음 push했다.

### 변경 내용

| 항목 | 파일 |
|------|------|
| git commit 직전 worklog 작성 유도 훅 (유저 레벨) | `~/.claude/hooks/worklog-on-git.sh` |
| 훅 등록 (PreToolUse: Bash) | `~/.claude/settings.json` |
| Stop 훅 검증용 로그 코드 제거(원복) | `.claude/hooks/lint-build-test.sh` (net 변경 없음) |

### 핵심 구현 포인트

- **훅은 스킬을 직접 실행 못 함** → `PreToolUse`에서 `permissionDecision: deny` + reason으로 Claude에게 worklog 실행을 *유도*하는 방식 채택.
- **commit 전(PreToolUse)이 핵심**: 작업일지를 그 커밋에 포함시키려면 push 후가 아니라 commit 직전에 유도해야 함.
- **무한 루프 방지**: 오늘자 worklog가 이미 스테이징(`git diff --cached`)돼 있으면 커밋 통과. 첫 시도 차단 → worklog add → 재커밋 시 통과.
- **false positive 교정**: `*"git commit"*` 단순 매칭은 `echo "git commit"`까지 막음. `&& ; |`·개행으로 조각낸 뒤 각 조각이 `git commit`으로 시작할 때만 매칭. (단, sed는 따옴표를 모르므로 문자열 인자 안의 `&& git commit`은 여전히 오탐 — 실사용 영향은 작음)

### 검증

- 훅 4케이스 테스트(파일 기반 러너): 실제 commit→deny, `echo`→allow, `git status`→allow, `npm test && git commit`→deny 모두 PASS.
- 실커밋 통과 확인: 작업일지 스테이징 상태에서 `git commit` 시 차단 없이 통과 (`7a86754`).
- 원격 push 성공: `9691689..7a86754 main -> main` (github.com/sseonmo/ai_practice_part1_ch1).
